# Signals

Watch for these signals and respond immediately.

| Signal | Meaning | Response |
|---|---|---|
| "I'm not sure this is the right scope" | Scope drift or over-specification | Return to outline locking |
| Unsupported code claim | Guessing | Read the code before proceeding |
| Same decision resurfaces | Decision not locked hard enough | Add or restate locked decision |
| Finding contradicts design doc | Possible upstream gap | Stop and route to design owner |
| Prep candidate fails a gate | Cosmetic or feature work | Reject or promote to feature |
| "And also" proposal | Multiple tickets in one | Split or drop weaker scope |
| Body-level code needed to justify interface | Interface is unclear | Rework interface or ask for deeper scope |
| Linear ticket content requested mid-spec | Premature packaging | Finish working docs first |
| User asks implementation question during design | Phase confusion | Clarify that design/scoping is not coding |
